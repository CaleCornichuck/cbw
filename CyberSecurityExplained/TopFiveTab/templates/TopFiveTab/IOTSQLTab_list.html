SQL and IOT




The Structured Query Language, or SQL, is the standard language used to manage a database, it includes the commands to update, retrieve, remove and organize the data in a database. The data in a database is stored in relational tables, which are made from a series of rows and columns. As the size of a table or database grows it quickly becomes impossible to use without SQL.




The Internet of Things, or IOT, refers to the physical objects around us that have some way of exchanging data. Examples of this would be smart devices that can be controlled remotely like security cameras set around a property, or a thermostat controlled from a phone. The current number of IOT devices has exceeded 12.3 billion as of 2021.




How is SQL used in cyberattacks?

How is IOT used in cyberattacks?


One of the reasons SQL injection attacks are so common, is that many websites have user logins. This requires storing user credentials in a database and being able to access them reliably.


If the websites database is not configured correctly, any user of the website can enter SQL commands to the login or url.


The login is already running an SQL command to verify if the input is a valid user in the database. The SQL code that the malicious actor injects changes the normal SQL command to alter it's intended function.


This alteration could make the SQL database return other users credentials or simply delete the entire database. SQL databases have the capability of security, by parameterising the possible commands allowed, but most companies use outdated software and configurations.

One of the main reasons IOT devices are a security risk is due to the complexity of the systems involved. Every one of the devices use network connections to receive and transmit data necessary to its function. Essentially, each one of these connections represent a possible area of attack to not just the owners, but the companies managing the data.


IOT devices are particularly vulnerable to becoming bots in a botnet, allowing the attacker to perform attacks such as DDOS. Any device compromised in this way is not just liable to be involved in other attacks, but cannot be trusted to handle personal data.


When an IOT device is compromised it can be subject to Man-In-The-Middle attacks. These attacks work by intercepting messages between two parties, such as building security devices and the company that manages them. This could be used to silence or trigger alarms at will or even remotely hack a vehicle and drive it away.
