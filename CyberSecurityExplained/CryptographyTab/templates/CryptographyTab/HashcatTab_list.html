Hashcat

Hashcat is a password recovery tool that uses hashes to decrypt data. While it is popular for testing security systems and recovering user information, many use it for more nefarious purposes, such as stealing passwords. It is an easily available tool that allows a user to brute force any passwords using hashes.



How Hashcat Works

Hashcat uses the hashed password to compare to a table or dictionary it has to find and crack the password. There are many different ways it can do this, such as:

  • Brute-Force Attacks

    • When given a set number of characters, Hashcat iterates through every single combination of upper-case letters, lower-case letters, and numbers. Brute-Force Attacks have since been replaced by Mask Attacks, which are almost the same, but are more specific, which reduces the amount of combinations used, and the time taken to find the password.


  • Dictionary Attacks

    • Given a text file, Hashcat reads through every line, and tries each one as a password. It is another common way to guess passwords, and also very simple.


  • Combinator Attack

    • Given a dictionary, Hashcat will concatenate each word to every other word, making a new list of password candidates. For example, if there are 3 words, such as pass, word, and cat, one of the candidates would be passcat.


  • Hybrid Attack

    • A Hybrid Attack is a Combinator Attack, with one side a dictionary, while the other is a Brute-Force Attack. The Brute-Force password is combined with each word in the dictionary.


  • Association Attack

    • Hashcat uses pieces of information that could have an influence on the password, such as a username, to attack a specific hash.
